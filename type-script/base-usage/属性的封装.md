## 为啥要进行属性的封装
* 1： 对象中的属性任意被修改，会导致对象中的数据变得非常的不安全
```
class Person {
    1: ts 可以在属性前面添加属性的修饰符
        public  : 默认值，在任意地方可以被修改
        private : 私有属性，只能在类内部进行访问和修改
                  通过在类的内部添加方法使得私有属性可以被外部访问。
      protected : 受保护的属性，只能在当前类和当前类的子类中访问或者修改。
      peotected num: number;
      constructor(params: number) {
          this.num = params;
      }
      直接将属性修饰符写在构造器的参数位置
       constructor(public params1: number,public params2: string) {
          this.num = params;
      }
    2: ts 中使用getter的方法
      get name(){
          return this.name
      }
      set name(value:string){
          this.name = value
      }

}

class Tom extends Person {
    print(){
        console.log(this.num) 
    }

}
const littleTom = new Tom(num: 123)
littleTom.num = 33   报错： 因为protected属性，只能在子类中进行访问和修改
```
